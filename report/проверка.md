### 2.2 Обмен файлами

**Вопрос 1:** Какая команда скопирует папку stepic на сервер?

![2.2.1](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.2%20Обмен%20файлами/1%20задание/2.2.1.png)

**Ответ:** `scp -r stepic username@server:~/`

**Пояснение:** `scp` — команда копирования по SSH, опция `-r` нужна для рекурсивного копирования папок.

---

**Вопрос 2:** Что делать, если `apt-get install` не может найти пакет?

![2.2.2](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.2%20Обмен%20файлами/2%20задание/2.2.2.png)

**Ответ:**
- `sudo apt-get update`
- Проверка интернет-соединения

**Пояснение:** `apt-get update` обновляет список пакетов. Отсутствие интернета также мешает скачиванию.

---

**Вопрос 3:** Для чего можно использовать Filezilla?

![2.2.3](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.2%20Обмен%20файлами/3%20задание/2.2.3.png)

**Ответ:**
- Для просмотра содержимого директорий на своем компьютере
- Для копирования файлов со своего компьютера на сервер
- Для копирования файлов с сервера на свой компьютер

**Пояснение:** Filezilla — FTP-клиент для обмена файлами. Установка и запуск программ не её функции.

### 2.4 Контроль запускаемых программ

**Вопрос 1:** Какие программы показывает `jobs` после операций?

![2.4.1](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.4%20Контроль%20запускаемых%20программ/1%20задание/2.4.1.png)

**Ответ:** Только о program2 и program3

**Пояснение:** `fg %1` с последующим `Ctrl+C` завершает program1. `fg %2` с `Ctrl+Z` приостанавливает program2. `jobs` показывает program2 и program3.

---

**Вопрос 2:** Одинаковые ли идентификаторы в jobs, top и ps?

![2.4.2](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.4%20Контроль%20запускаемых%20программ/2%20задание/2.4.2.png)

**Ответ:** Одинаковые только у ps и top

**Пояснение:** `ps` и `top` показывают PID процесса. `jobs` показывает номер задачи в текущей сессии терминала.

---

**Вопрос 3:** Как мгновенно завершить остановленный процесс?

![2.4.3](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.4%20Контроль%20запускаемых%20программ/3%20задание/2.4.3.png)

**Ответ:** `kill -9`

**Пояснение:** Сигнал SIGKILL (`-9`) немедленно завершает процесс без возможности обработки.

---

**Вопрос 4:** Что произойдет при `kill` без опций для приостановленного процесса?

![2.4.4](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.4%20Контроль%20запускаемых%20программ/4%20задание/2.4.4.png)

**Ответ:** Процесс приступит к завершению, как только будет продолжен

**Пояснение:** Приостановленный процесс не может обработать сигнал. `kill` без опций отправляет SIGTERM, который будет обработан при возобновлении.

### 2.6 Менеджер терминалов tmux

**Вопрос 1:** Что будет при наборе `fg` во второй вкладке?

![2.6.1](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.6%20Менеджер%20терминалов%20tmux/1%20задание/2.6.1.png)

**Ответ:** Терминал сообщит, что нет процесса для запуска в fg

**Пояснение:** Каждая вкладка — отдельная сессия. Процесс, приостановленный в одной вкладке, не виден в другой.

**Ответ:** tmux завершит работу

**Пояснение:** Закрытие последней вкладки завершает сессию tmux.

![2.6.3](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.6%20Менеджер%20терминалов%20tmux/3%20задание/2.6.3.png)

**Вопрос 6:** Верные утверждения о разделении вкладок в tmux.

![2.6.6](https://github.com/slavagoltzoni-boop/linux-course-report/blob/main/screenshots/2%20%20Работа%20на%20сервере/2.6%20Менеджер%20терминалов%20tmux/6%20задание/2.6.6.png)

**Ответ:**
- Команды разделения действуют только в текущей вкладке
- При вертикальном разделении горизонтально разделенной вкладки получится 3 части
- По половинкам можно перемещаться при помощи Ctrl+B и стрелочек
- Вкладку можно разделять несколько раз
- Можно закрыть одну из частей через Ctrl+B и x

**Пояснение:** Tmux поддерживает множественное разделение вкладок и навигацию между частями.
